# WeChat-Insight-Skill

一个面向 **Antigravity IDE** 的 AI Skill，专门用于写微信公众号文章。

把你的想法、项目、技术内容，变成适合公众号传播的推文——有人味、有结构、不 AI 腔，并且默认按“可直接发布”的标准输出完整草稿。

---

## 它能做什么

- **技术科普**：把复杂的技术概念翻译成泛读者能看懂的大白话
- **产品推广（软文）**：用独立开发者视角植入产品，自然不违和
- **教程干货**：清单式结构，适合收藏转发
- **行业观察**：有观点、有态度，不做中立的信息搬运工
- **技术文档 → 公众号**：把已有的博客/文档改写成手机竖屏友好的推文
- **传播增强**：围绕粘性、流通、扩散、易读四个指标，补足收藏点、转发理由和扫读结构

默认风格是**独立开发者调性**：有人味但不油腻，有观点但不煽动，像一个靠谱朋友在咖啡馆聊他最近做的东西。

## 安装方式

### 方式一：直接复制到你的 Skills 目录

```bash
# 克隆到你的 Antigravity Skills 目录
git clone https://github.com/YOUR_USERNAME/wechat-article-writer-skill.git \
  ~/.antigravity/skills/wechat-article-writer
```

### 方式二：手动放置

1. 下载或克隆本仓库
2. 将整个文件夹放到你的 Antigravity IDE Skills 目录下（通常是 `~/.antigravity/skills/` 或项目内的 `.agent/skills/`）
3. 重启 IDE，Skill 会自动被识别

## 使用方式

在 Antigravity IDE 对话框中，用自然语言触发即可：

```
帮我写一篇公众号，介绍我用 Python 做的自动剪视频工具
```

```
把这篇技术博客改写成公众号版本，读者是泛技术爱好者
```

```
写一篇推广 Seedance API 的软文，用独立开发者调性
```

Skill 会优先根据上下文推断文章类型、目标读者和核心目标；只有方向会明显跑偏时才追问。默认输出：

- 使用假设（文章类型、目标读者、核心目标、作者调性）
- 5 个备选标题
- 40-80 字公众号摘要
- 完整正文（含小标题、加粗重点、**配图直接生成并嵌入正文**——主动判断每段内容的可视化价值，有的地方配流程图/原理图，有的地方配橘猫叙事插图，没有可视化价值的段落不强行配）
- 朋友圈/社群转发文案
- 发布前检查项（只列需用户补齐的链接和数据来源）

## Skill 内容概览

| 章节             | 内容                                            |
| ---------------- | ----------------------------------------------- |
| 一、先想清楚     | 动笔前必须明确的 4 个问题                       |
| 二、标题         | 6 种套路 + 独立开发者标题写法 + 红线            |
| 三、传播四力     | 粘性、流通、扩散、易读的生成和检查方法          |
| 四、开头         | 前 3 行决定生死，禁止和有效写法对比             |
| 五、作者调性     | 三大流派定义 + 去 AI 味完整规则                 |
| 六、正文结构     | 排版铁律 + 5 种结构模板 + 配图生成工作流（叙事插图 / 流程图 / 封面图）|
| 七、技术内容翻译 | 如何在口语化和技术深度之间找平衡                |
| 八、产品推广规范 | 软文写法的额外要求                              |
| 九、结尾与 CTA   | 4 种结尾目标 + 外链处理（微信特有坑）+ 转发文案 |
| 十、合规与风险   | 极限词、诱导分享、版权等平台红线                |
| 十一、封面图     | 尺寸规格、安全区、三套调性风格 + prompt 模板    |
| 十二、终极审查   | 13 条 Final Check 清单                          |

图片生成的详细规范按类型拆到各 reference 文件，所有文件共享一套全局视觉设计系统：

| 文件 | 层级 | 负责内容 |
| ---- | ---- | -------- |
| [`references/design-system.md`](./references/design-system.md) | 基础层 | 全局配色、线条、留白、标注规范，所有图片必须继承 |
| [`references/cartoon-illustrations.md`](./references/cartoon-illustrations.md) | 专属层 | 叙事 / 比喻 / 情绪类插图，固定使用**橘猫** IP 主角 |
| [`references/flowchart-style.md`](./references/flowchart-style.md) | 专属层 | 流程图 / 架构图 / 原理示意图，含四种图表类型模板；橘猫可巧妙融入（需有叙事角色，不贴角落装饰）|
| [`references/cover-image.md`](./references/cover-image.md) | 专属层 | 封面图，含独立开发者 / 推广种草 / 硬核技术三套风格 |

只有需要实际生成对应图片时才读取对应文件。生成前必须先读取 `design-system.md`，再叠加专属层的风格词。

## 去 AI 味内置规则

本 Skill 内置了完整的去 AI 腔规则，不依赖外部 Skill：

- 删填充短语（\"此外\"\"综上所述\"\"值得注意的是\"）
- 打破三段式排比结构
- 混合句子节奏，避免等长句
- 去宣传腔（\"革命性的\"\"颠覆性的\"\"强大的\"）
- 盘古之白（中英文之间加空格）
- 完稿前 6 条快速自检清单
