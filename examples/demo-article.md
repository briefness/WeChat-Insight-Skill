## 正文

剪一条 30 秒的产品视频，找素材、调色、配乐、加字幕，一个下午没了。

上个月接了个需求，要出 10 条风格统一的短视频，我直接开始想能不能让这件事自动化。

**结论是可以，但 AI 只解决了两成问题，剩下八成是在处理各种失败。**

这篇把整个流程拆开聊聊，代码放在文末。

---

### 整体思路

流程很简单，四步：

1. 输入一句话描述（\"海边日落，慢慢推进，暖色调\"）
2. 用 LLM 拆解成分镜脚本
3. 逐帧调用 Seedance 视频生成 API
4. 用 FFmpeg 拼接成片 + 混音

说起来简单，实际跑通花了两天。主要时间不在生成，在容错。

---

### 分镜脚本：让 LLM 做导演

第一步是把用户输入拆成结构化分镜。

```python
def parse_storyboard(prompt: str) -> list[Scene]:
    response = client.chat(
        model="gemini-2.0-flash",
        messages=[{"role": "user", "content": STORYBOARD_SYSTEM + prompt}]
    )
    return json.loads(response.content)
```

系统提示词里规定了每个分镜的格式：时长、运镜类型、画面描述、情绪关键词。LLM 输出 JSON，我直接反序列化。

这里有个坑：LLM 偶尔会输出不合法的 JSON（多个反引号、带注释）。加了一层用正则提取 JSON 块的兜底逻辑，成功率从 80% 拉到 99%。

---

### 视频生成：并发 + 重试

每个分镜独立调用 Seedance API 生成，任务之间没有依赖，所以可以并发。

```python
async def generate_scenes(scenes: list[Scene]) -> list[Path]:
    tasks = [generate_single(s) for s in scenes]
    return await asyncio.gather(*tasks)
```

并发带来一个新问题：API 限速。加了指数退避重试，最多重试 3 次，超过就跳过这个分镜、用黑场填充。

宁可成片有一段黑屏，也不让整个任务卡死。

---

### FFmpeg 拼接：最容易踩的坑

把所有片段拼起来，用 FFmpeg 的 `concat` 滤镜。

**这里有个不起眼但不做就翻车的细节：FFmpeg 拼接要求每段视频都有音轨，哪怕是静音的。** 如果某段 AI 生成的视频没有音频流，整个拼接命令会静默失败，输出一个 0 字节文件。

解决方法是在拼接前统一给每段视频混入一条静音轨：

```python
def ensure_audio_track(video_path: Path) -> Path:
    output = video_path.with_suffix(".normalized.mp4")
    subprocess.run([
        "ffmpeg", "-i", str(video_path),
        "-f", "lavfi", "-i", "anullsrc",
        "-shortest", "-c:v", "copy", str(output)
    ], check=True)
    return output
```

一行命令，省了我半天调试时间。

跑一次完整流程大概 2-3 分钟，出一条 30 秒的成片。

### 局限性（诚实说）

几个现在还没解决的问题：

- **分镜之间的画面连贯性**：每段独立生成，人物/场景在段落之间会跳变
- **长视频成本**：10 个分镜 × API 调用费用，比剪辑软件贵
- **中文字幕**：FFmpeg 的中文字体配置在不同系统上行为不一致，还没统一处理

这中间的距离，就是\"玩具\"和\"工具\"的距离。我目前用在批量产出场景，对连贯性要求不高的地方。
